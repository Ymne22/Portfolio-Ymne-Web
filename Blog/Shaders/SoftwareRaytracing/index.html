Overview

This project explores a real-time software-based ray tracer built from scratch in Unity, leveraging Compute Shaders instead of relying on hardware-accelerated ray tracing APIs (like RTX).

It implements a custom Bounding Volume Hierarchy (BVH) for acceleration, recursive light transport including reflections and indirect lighting, and a procedurally generated sky system. This approach allows for ray tracing capabilities on a wider range of hardware that may not support native DXR or Vulkan Ray Tracing.

System Architecture

The engine is divided into several core components that work together to translate Unity scene data into a format that the GPU can trace efficiently.

1. Scene Data Conversion

The RayTracingController collects all scene meshes and materials, flattens geometry into a single structure, extracts textures, and packs material data into compute buffers. Each mesh is stored with its vertices, normals, tangents, UVs, and triangle indices.

2. BVH Construction

A Surface Area Heuristic (SAH) based binary BVH is constructed on the CPU. This acceleration structure significantly reduces the number of intersection tests by grouping primitives into a hierarchical volume.

3. Compute Shader Dispatch

At render time, the ray tracing shader is dispatched with a thread group per screen tile. Each thread generates a primary ray from the camera, traverses the BVH for intersections, and computes the light transport (Direct, Indirect, Reflections, and AO).

4. Temporal Accumulation

To combat the noise inherent in undersampled ray tracing, the system implements a temporal accumulation buffer. This blends the current frame with previous frames using motion vectors and reprojection to achieve a clean, noise-free image over time.

Technical Parameters

The system is highly configurable via the Unity Inspector, allowing for a balance between performance and visual quality:

  • Resolution Scale: Full, Half, or Quarter rendering resolution.
  • Max Bounces: Controls the depth of recursive reflections (1–32).
  • Samples Per Pixel: Per-frame sampling count for stochastic effects.
  • Temporal Capture: Enables frame blending to reduce noise.
  • Raster Fallback: Blends ray-traced results with the standard raster pass.

Limitations

Note: As a software-based experimental project, there are several limitations to be aware of:

  • Supports Unity Built-in Render Pipeline (BRP) only.
  • No caustics or energy conservation (it's not a full path tracer).
  • Basic transparency support (alpha only).
  • Performance scales linearly with triangle count and bounce depth.
  • No support for dynamic mesh deformation (skinned meshes).

Usage Instructions

To integrate the software ray tracer into your project:

  1. Add the RayTracingController script to your Main Camera.
  2. Assign the RayTracing.compute shader in the component's inspector.
  3. Configure your scene objects with the appropriate material settings.
  4. Enable ExecuteInEditMode to see real-time ray-traced results directly in the scene view.
