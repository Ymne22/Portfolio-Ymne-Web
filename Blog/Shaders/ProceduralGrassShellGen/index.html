Overview

The Procedural Grass Shell Generator is a specialized shader system designed to render dense, lush grasslands without the overhead of thousands of individual mesh blades. By utilizing a Shell Texturing approach, the shader "stacks" virtual layers to create volume, depth, and complex motion.

This implementation is fully procedural, meaning it doesn't rely on pre-baked noise textures or UV maps. Instead, it synthesizes grass patterns in real-time using mathematical noise functions, allowing for infinite scalability and zero memory footprint for textures.

Shell Texturing Stack

Unlike traditional geometry-based grass, shell texturing works by rendering multiple "slices" of a volume. In this shader, we use a Parallax Marching approach inside a single pass to determine the intersection of the view ray with virtual grass blades.

Note: While many shell shaders use multi-pass rendering (one pass per shell), this implementation uses a screen-space ray marching technique in a single shader pass, which is significantly more efficient for modern GPUs.

  • Virtual Height: Each sample represents a different height layer from the ground (0.0) to the tip (1.0).
  • Alpha Thresholding: As we go higher, the blades become thinner, following a procedural curve to simulate natural tapering.

Wind Waves & Gusts

To achieve believable motion, the shader implements a multi-layered wind system that combines smooth sine waves with multi-octave noise (FBM).

  • Primary Waves: Smooth, rolling sine-based displacement derived from world position. This creates the "big" movement across the field.
  • Gust Intensity: High-frequency noise that modulates the wind strength over time, creating "bursts" of motion that feel unpredictable.
  • Triplanar Compatibility: The wind is calculated in world-space and projected back into tangent-space, ensuring it works correctly on hills, slopes, and vertical surfaces.

Parallax & Refinement

Standard shell shaders often look like "pancakes" when viewed from steep angles. To solve this, we implement a Linear Search + focused Binary Refinement phase.

The shader first marches through the shells with a coarse step to find a potential hit. Once a hit is detected, it performs _ParallaxRefinement steps between the current and previous shell to pinpoint the exact intersection.

This hybrid approach allows us to achieve high-fidelity silhouettes with only 8-16 coarse steps, keeping the instruction count low while maintaining visual quality.
