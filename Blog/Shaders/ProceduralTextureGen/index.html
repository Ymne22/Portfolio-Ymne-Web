Overview

The Procedural Texture Generator is a comprehensive shader solution for Unity's Built-in Render Pipeline. It allows for the creation of high-fidelity environmental textures entirely through mathematical logic and noise shaders, eliminating the need for traditional image assets.

This approach provides infinite variation, perfect tiling, and the ability to change material types (Brick, Wood, Stone) dynamically through a single set of uniform parameters.

Material Generation

The shader uses a modular synthesis approach. By defining different grid logic for each material type, we can switch between architectural patterns without changing the underlying geometry.

  • Brick & Tile: Uses a staggered grid with customizable grout lines.
  • Wood Planks: Implements randomized grain noise and density-based visibility for "wear".
  • Stone & Sand: Utilizes multi-layered Voronoi and Perlin noise to simulate organic surface irregularities.
HLSL 
// Procedural Plank Grid
float2 plankUV = uv * _PlankTiling;
float plankID = floor(plankUV.y);
plankUV.x += hash(plankID) * _PlankOffset; // Randomized stagger

Parallax Depth Mapping

To give the procedural surfaces a tactile, 3D feel, the shader implements Parallax Mapping. It uses the generated heightmask to distort the texture coordinates based on the view direction, effectively "peering" into the crevices of bricks or the gaps between wood planks.

Weathering & Aging

Surfaces can be procedurally aged using the _Dirtiness and _Weathering sliders. This logic samples a global noise field masked by the surface cavities (found via the heightmap derivative), ensuring that dirt accumulates realistically in the gaps and depressions.

Included Features

  • Shell Texturing Support: Optimized for high-density shell-based rendering for fur or complex depth.
  • Randomized Detailing: Automatic variation per-instance to avoid visible tiling patterns.
  • Dynamic Masking: Separated controls for base color, grout/gap color, and "wear" overlays.
