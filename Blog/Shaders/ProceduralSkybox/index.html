Overview

This procedural skybox shader for Godot 4.x implements a comprehensive atmospheric rendering system. Unlike static HDRI skyboxes, this shader computes sky colors dynamically based on sun position, simulates realistic light scattering, and renders volumetric clouds using raymarching.

The system combines Rayleigh scattering for sky tint, Mie scattering for sun glow, and a multi-pass volumetric cloud system that reacts to global illumination.

Atmospheric Physics

Real-world sky color is the result of light interacting with particles in the atmosphere. We simulate two main types of scattering:

  • Rayleigh Scattering: Interaction with small air molecules. It scatters shorter wavelengths (blue) more easily, giving the sky its color.
  • Mie Scattering: Interaction with larger particles like dust or water droplets. It causes the bright glow around the sun and the hazy horizon.

We use ray_sphere_intersect to find the boundaries of our atmospheric volume, usually defined as a shell between planet_radius and atmosphere_radius.

Calculating Scattering

To compute the color at any point, we march along the view direction and integrate the scattering at each step. For each sample point, we also check the occlusion from the sun (optical depth) to account for extinction.

GDScript Shader 
float rayleigh_phase(float cos_theta) {
    return (3.0 / (16.0 * PI)) * (1.0 + cos_theta * cos_theta);
}

float henyey_greenstein_phase(float cos_theta, float g) {
    float g2 = g * g;
    return (1.0 / (4.0 * PI)) * ((1.0 - g2) / pow(1.0 + g2 - 2.0 * g * cos_theta, 1.5));
}

Celestial Bodies

The shader handles the transition between day and night seamlessly. Procedural stars are generated using a 3D grid with value noise for position and twinkle. The Sun and Moon are rendered as procedural discs, with the moon supporting a texture map for detailed craters.

Volumetric Clouds

The most complex part of the shader is the volumetric cloud system. It uses Raymarching to step through a cloud layer defined between two altitudes. At each step, we sample a density function powered by multi-layered 3D noise (FBM).

GDScript Shader 
float get_cloud_density(vec3 pos, float detail_fade) {
    vec3 sample_pos = (pos + noise_seed_offset + wind_offset) * noise_scale;
    float base_shape = fbm(sample_pos, 4);
    
    // Mask by coverage
    base_shape = clamp((base_shape - coverage) / (1.0 - coverage), 0.0, 1.0);
    
    // Add curl noise for billowy edges
    vec3 curl = curl_noise(sample_pos * curliness) * 0.1;
    base_shape = clamp(base_shape + curl.x, 0.0, 1.0);
    
    return base_shape * height_factor * density;
}

Cloud Lighting & Self-Shadowing

To make clouds look three-dimensional, we use the Beer-Lambert Law for light absorption. For each cloud fragment, we take additional light samples towards the sun to calculate how much light reaches that point.

We also implement the Powder Effect, which darkens the edges of clouds where scattering is highest, simulated by pow(1.0 - energy, 2.0).

Optimization & Dithering

Raymarching many steps is expensive. To optimize:

  • Adaptive Light Steps: We reduce light sampling quality at further distances.
  • Blue Noise Dithering: We jitter the ray start position using blue noise to turn banding artifacts into subtle grain, which is much more pleasing.
  • Detail LOD: Detail noise is faded out at long distances to reduce sampling overhead.
