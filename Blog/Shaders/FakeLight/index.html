Overview

The Fake Light System is a high-performance lighting solution for Unity's Built-in Render Pipeline. Instead of using standard Unity Point Lights which can be expensive in large quantities, this system utilizes a custom shader and a C# controller to simulate localized radial lighting with advanced features like volumetric halos and raymarched shadows.

Note: This shader uses Blend DstColor One logic. It requires at least a small amount of ambient or directional light to be present in the scene. If a surface is pitch black (0,0,0), the fake light will have nothing to brighten and will disappear. Always use a low-intensity ambient light or environment skybox.

The system works by rendering a procedurally generated low-poly sphere and calculating the lighting influence precisely on the geometry it intersects.

Why use this for Forward Rendering?

Unity’s Forward Renderer has a strict limit on the number of real-time pixel lights (typically 4 or 8). Exceeding this limit forced the engine to use vertex lighting, which produces artifacts, performance issues, or additional rendering passes. Because this system is a custom geometric effect, it does not count toward Unity's light limit, allowing you to fill your scenes with hundreds of dynamic lights without performance hitching.

World Position Reconstruction

To apply lighting to the scene geometry, the shader reconstructs the world position of every pixel using the camera's depth texture and a precomputed ray direction.

HLSL 
float3 rayDir = normalize(i.rayDir);
float viewZ = dot(rayDir, -UNITY_MATRIX_V[2].xyz);
float3 sceneWorldPos = _WorldSpaceCameraPos + rayDir * (linearDepth / viewZ);
float3 sceneObjPos = mul(unity_WorldToObject, float4(sceneWorldPos, 1.0)).xyz;

Screen-Space Raymarched Shadows

The most advanced feature is the inclusive shadow system. It performs short-range raymarching from the scene surface towards the light center within the screen space, using Bayer Dithering or Interleaved Gradient Noise to achieve soft, performant shadows.

Tip: Enable TAA Mode to use high-frequency temporal noise, which provides perfectly smooth shadows when used with a Temporal Anti-Aliasing solution.
